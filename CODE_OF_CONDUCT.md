# Code of Conduct

## Our Standard

We want Coredot Editor to be useful to people building serious document and AI products. Project spaces should stay professional, respectful, and focused on the work.

Examples of encouraged behavior:

- Use welcoming and inclusive language.
- Be direct about technical concerns without making them personal.
- Give actionable feedback.
- Respect different levels of experience.
- Assume good intent, and ask for clarification when needed.

Examples of unacceptable behavior:

- Harassment, insults, threats, or discriminatory language
- Sexualized language or imagery
- Publishing private information without permission
- Repeated off-topic disruption
- Bad-faith engagement with maintainers or contributors

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or restrict participation when behavior harms the project community.

If you need to report a conduct issue, contact a maintainer privately. Include relevant links, screenshots, or context where possible.

## Scope

This code of conduct applies to project repositories, issue trackers, pull requests, discussions, and other project-managed spaces.
